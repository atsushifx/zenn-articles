## はじめに

本記事では、関数型言語の初心者のために Racket 言語における基本データ型について解説します。

Racket には多様なデータ型がありますが、本記事では数値型、ブーリアン型、文字型、文字列型、シンボル型、Pair 型、リスト型、Void、undefined について説明します。

## Racketとは

Racket は、LISP[^1]系のプログラミング言語であり、Scheme 言語を基盤にした関数型言語[^2]です。LISP は、リストを操作できるプログラミング言語で、現代の関数型言語の基礎となっています。

[^1]: LISP: LISt Processing の略語で、リストを操作できるプログラミング言語。
[^2]: 関数型言語: プログラムを数学的な関数の集合として考えるプログラミングパラダイムの 1つ。副作用を排除することで、プログラムの信頼性や並列処理の実現が可能となる。

## Racketにおけるデータ

データは、Racket で扱うことができるあらゆる値のことを指します。
そして値を宣言したときに、Racket が適切なデータ型に型付けします。
整数、文字列、リスト、ハッシュテーブル、構造体、関数などがデータの例です。
データは不変であり、値自体は変更できません。Racket のデータは、Scheme と同じく、単純な値として扱われます。

## Racketのおける基本データ型

### 数値型

数値型には、整数型、有理数型、実数型があります。

#### 整数型

整数型は、正負の整数および 0 を表現するために使用します。整数型は、integer と呼ばれます。以下のように、整数値を定義できます。

``` racket
(define my-integer 42)

```

上記の例では、変数 my-integer に `42` を代入しています。整数型には、以下のような算術演算子を使用できます。

- `+` : 加算
- `-` : 減算
- `*` : 乗算
- `/` : 除算
- `modulo` : 剰余演算

#### 有理数型

有理数型は、分数を表現するために使用します。有理数型は、rational と呼ばれます。以下のように、有理数値を定義できます。

``` racket
(define my-rational 3/4)

```

上記の例では、変数 my-rational に 3/4 を代入しています。有理数型には、以下のような算術演算子を使用できます。

- `+` : 加算
- `-` : 減算
- `*` : 乗算
- `/` : 除算

#### 実数型

実数型は、小数点以下の桁数を持つ数を表現するために使用されます。実数型は、float または real と呼ばれます。以下のように、実数値を定義できます。

``` racket
(define my-float 3.141592)

```

上記の例では、変数 my-float に `3.141592` を代入しています。実数型には、以下のような算術演算子を使用できます。

- `+` : 加算
- `-` : 減算
- `*` : 乗算
- `/` : 除算

### ブーリアン型

ブーリアン型は、真(`#t`) または偽(`#f`)の値をとるデータ型です。
ブーリアン型は、 boolean と呼ばれます。数値の比較など、条件の成否などを表すために使用します。

たとえば、以下のように、ブーリアン値を定義できます。

``` Racket
(define my-boolean #t)

```

上記の例では、変数 my-boolean に #t を代入しています。
ブーリアン型には、ブーリアン型には、以下のような論理演算子が使えます。

- `not` : 否定
- `and` : 論理積
- `or`  : 論理和
- `xor` : 排他的論理和

### 文字型

文字型は、Unicode 文字を表現するために使用されます。文字型は、 char と呼ばれます。以下のように、文字を定義できます。

``` Racket
(define my-char #\a)

```

上記の例では、変数 my-char に文字 a を代入しています。文字型には、以下のような比較演算子を使用できます。

- `char=?`  : 文字が等しいかどうかを比較する
- `char<?`  : 文字の大小関係を比較する
- `char>?`  : 文字の大小関係を比較する
- `char<=?` : 文字の大小関係を比較する
- `char>=?` : 文字の大小関係を比較する

### 文字列型

文字列型は、文字列を表現するために使用されます。文字列型は、二重引用符で囲まれた文字列です。文字列型は、 string と呼ばれます。以下のように、文字列を定義できます。

``` Racket
(define my-string "hello")

```

上記の例では、変数 my-string に文字列 hello を代入しています。文字列型には、以下のような操作ができます。

- `string-append` : 文字列を連結する
- `string-length` : 文字列の長さを取得する
- `string-ref`    : 文字列の指定した位置にある文字を取得する

### シンボル型

シンボル型は、文字列と同様に名前を表現するために使用されます。
シンボル型は、通常、値として使用されることはありません。その代わり、変数や手続きの名前のために使用されます。
シンボル型は、'<シンボル名>で表されます。たとえば、'apple はシンボル apple を表します。

### ペア型とリスト型

ペア型とリスト型は、LISP 処理系の基本的な概念を示す型です。
ペア型は、2つの要素を持つペアを表していて、`cons` を使用して作成します。作成したデータをコンスセル[^3]と呼びます。リスト型は、コンスセルを複数つなげて作成するデータ型であり、複数のデータをひとまとまりにして扱うことができます。

たとえば、(cons 1 2)は下記のように、数値 1、2 の要素をもつコンスセルを作成します。

``` Racket : (1 . 2)
    +---+---+
    | 1 | 2 |
    +---+---+
```

このコンスセルの最初の要素を car、次の要素を cdr と呼びます。

`(list 1 2 3)`は、数値 1、2、3 の要素をもつリストを作成します。
これを図でしめすと、

``` Racket : (1 2 3)
    +-----+------+     +-----+------+     +-----+-----+
    |  1  |  o---+---->|  2  |  o---+---->|  3  | '() |
    +-----+------+     +-----+------+     +-----+-----+
```

このように、Racket ではリストはコンスセルのつながりとして実現されます。
リストの最後には、エンドマークとして空のリスト`'()`が入ります。

すなわち、コンスセルの cdr 部にコンスセルを入れることでリストを作成できます。
`(cons 1 (cons 2 (cons 3 '())))` として作成されるデータは、(list 1 2 3) と同じものになります。

[^3]: コンスセル: `cons`手続きによって作成されるデータで、リストの構成要素

### Void型

Void 型は、値を持たない型です。手続きが値を返す必要がない場合に使用されます。

たとえば、下記のコード、

``` Racket: void-procedure
(define void-procedure
    (void))

```

は、手続き`void-procedure`を定義しています。`void-procedure`は値を返す必要がないので、`void`を使用しています。

### undefined

`undefined`型は、未定義の値を表現するために使用されます。プログラム実行中に値が設定されていない変数を参照した場合、`undefined`値が返されます。

Racket では、`undefined`値を表す `undefined`シンボルが用意されています。undefined は、特定の値を持たないため、任意の型に属できます。

``` racket
(define x undefined)  ; x は undefined 値となる
(define y 1)
(display (+ x y))     ; undefined 値と 1 を足した結果は undefined

```
